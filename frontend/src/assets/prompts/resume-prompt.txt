Задача:
Сгенерируй профессиональное резюме на основе профиля пользователя, следуя структуре, описанной в person-schema.yaml, и учитывая лучшие практики составления резюме. Резюме должно быть четким, структурированным и адаптированным под желаемую позицию.

Входные данные:

Профиль пользователя (в приложенном yaml-файле).

Желаемая позиция (выбранную желаемую позицию, исключи нерелевантные выбранной позиции скиллы).

Требования к резюме:

Формат:

Используй Markdown для оформления.

Сохрани логическую структуру, но адаптируй её под стандарты резюме (например, разделы "Опыт работы", "Образование", "Навыки" и т.д.).

Персонализация:

Укажи желаемую позицию в заголовке резюме.

Подчеркни ключевые навыки и опыт, релевантные для желаемой позиции.

Структура резюме:

Контактная информация: Имя, телефон, email, дополнительные контакты (если есть).

Локация: Город, страна, готовность к переезду/удаленной работе/командировкам.

Цель (опционально): Краткое описание карьерных целей или профессионального профиля (1-2 предложения).

Навыки:

Группируй навыки по категориям (например, "Технические навыки", "Языки").

Для технических навыков укажи уровень (1-10) и дату последнего использования.

Для языков укажи уровень в формате "A1-C2".

Опыт работы:

Перечисли компании, должности и периоды работы.

Добавь описание задач и ключевых достижений (используй глаголы действия: "разработал", "оптимизировал", "увеличил").

Для достижений укажи метрики (например, "Увеличил производительность на 20%").

Упомяни используемые технологии (стек).

Образование: Укажи учебные заведения, степени, специальности и годы окончания.

Дополнительная информация:

Хобби (если релевантно).

Прочитанная литература (если имеет отношение к желаемой позиции).

Оптимизация:

Избегай избыточности.

Используй четкие и конкретные формулировки.

Сделай резюме легко читаемым (используй списки, абзацы, выделение ключевых моментов).

Пример вывода (Markdown):

markdown
# [Имя пользователя]  
**Желаемая позиция:** [Название должности]  

## Контактная информация  
- Телефон: [номер]  
- Email: [адрес]  
- LinkedIn/GitHub: [ссылки, если есть]  

## Локация  
[Город, Страна]  
- Готовность к переезду: [Да/Нет]  
- Удаленная работа: [Да/Нет]  
- Командировки: [Да/Нет]  

## Навыки  
### Технические навыки  
- [Навык 1]: Уровень [X/10], последнее использование: [дата]  
- [Навык 2]: Уровень [Y/10], последнее использование: [дата]  

### Языки  
- [Язык 1]: Уровень [C1]  
- [Язык 2]: Уровень [B2]  

## Опыт работы  
### [Компания 1]  
**Должность:** [Название]  
**Период:** [Дата начала] – [Дата окончания]  
- Задачи:  
  - [Описание задачи 1].  
  - [Описание задачи 2].  
- Достижения:  
  - [Достижение 1 с метриками].  
  - [Достижение 2 с метриками].  
- Технологии: [Стек технологий].  

## Образование  
### [Учебное заведение]  
**Степень:** [Название]  
**Специальность:** [Название]  
**Год окончания:** [Год]  

## Дополнительная информация  
- Хобби: [список].  
- Литература: [список, если релевантно].  
Примечание:
Если какие-то данные отсутствуют в профиле пользователя, пропусти соответствующий раздел или укажи "Не указано". Сфокусируйся на релевантности для желаемой позиции.

person-schema.yaml:
"
$schema: http://json-schema.org/draft-07/schema#
type: object
properties:
  person:
    type: object
    properties:
      name:
        type: string
      gender:
        type: string
        enum: [male, female, unknown]
      desiredPositions:
        type: array
        items: string
      contact:
        type: object
        properties:
          phone:
            type: string
            pattern: '^\+[0-9]{11}$'
          email:
            type: string
            format: email
        additionalProperties:
          type: string
          format: uri
        required: [phone, email]
      location:
        type: object
        properties:
          country:
            type: string
          city:
            type: string
          relocation:
            type: boolean
          remote:
            type: boolean
          business_trips:
            type: boolean
        required: [city, relocation, remote, business_trips]
      languages:
        type: array
        items:
          type: object
          properties:
            language:
              type: string
            level:
              type: string
              pattern: "^[A-C][1-2]$"
          required: [language, level]
      skills:
        type: array
        items:
          type: object
          properties:
            area:
              type: string
            name:
              type: string
            level:
              type: integer
              minimum: 1
              maximum: 10
            date:
              type: string
              format: date
          required: [area, name, level, date]
      education:
        type: array
        items:
          type: object
          properties:
            institution:
              type: string
            degree:
              type: string
            specialty:
              type: string
            year:
              type: integer
          required: [institution, degree, specialty, year]
      hobby:
        type: array
        items:
          type: string
      literature:
        type: array
        items:
          type: string
      experience:
        type: array
        items:
          type: object
          properties:
            company:
              type: string
            position:
              type: string
            from-date:
              type: date
            to-date:
              type: date
            tasks:
              type: array
              items:
                type: string
            stack:
              type: array
              items:
                type: string
            achievements:
              type: array
              items:
                type: object
                properties:
                  name:
                    type: string
                  initial_value:
                    type: number
                  final_value:
                    type: number
                  uom:
                    type: string
                    enum: [day, percent]
                  type:
                    type: string
                    enum: [increased, decreased]
                required: [name, initial_value, final_value]
          required: [company, position, from-date, tasks, stack, achievements]
    required:
      [
        name,
        gender,
        contact,
        location,
        languages,
        skills,
        education,
        experience,
      ]
required: [person]

"
